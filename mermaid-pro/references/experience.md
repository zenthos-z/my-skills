# Mermaid 踩坑经验 (Mermaid Experience Notes)

记录实际使用中遇到的语法陷阱和解决方案。持续更新。

---

## 花括号 {} 陷阱

### 问题
花括号 `{}` 是 Mermaid 的**保留字符**，用于：
- Subgraph 定义: `subgraph id["Name"]`
- 特殊形状: `A{{Rounded}}`
- 注释块

**直接出现在节点文本中会导致解析错误**。

### 踩坑案例

```javascript
// ❌ 错误 - 花括号直接出现在方括号内
M4_3[解析变量语法<br/>{{var}} / {{#var}}...{{/var}}]
// Error: Expecting 'SQE', got 'DIAMOND_START'

// ✅ 正确 - 用引号包裹整个文本
M4_3["解析变量语法 {{var}} / {{#var}}"]

// ✅ 更推荐 - 避免在节点中使用花括号，用描述性文字代替
M4_3["解析变量语法"]
```

### 解决方案优先级

1. **最佳**: 避免在节点文本中使用花括号，改用描述性文字
2. **次选**: 用双引号 `"..."` 包裹整个节点文本
3. **备选**: 使用其他符号替代，如 `「var」` 或 `<var>`

## HTML 换行标签

### 问题
`<br/>` 在某些渲染器中不工作或被转义。

### 解决方案
- 简化文本，避免换行
- 使用短文本，让布局自动处理

```javascript
// ❌ 避免
M1[使用<br/>--prompt 内容]

// ✅ 推荐
M1["使用 --prompt 内容"]
```

---

## 斜杠 / 在方括号内

### 问题
`[/text/]` 被解析为特殊形状语法（梯形）。

### 解决方案
用引号包裹或避免斜杠:

```javascript
// ❌ 错误 - 被解析为梯形
[/--input/]

// ✅ 正确
["--input 文件路径"]
```

---

## 经验总结 Checklist

遇到语法错误时检查：

- [ ] 节点文本是否包含 `{}` 花括号？
- [ ] 方括号内是否以 `/` 开头？
- [ ] 是否使用了 `<br/>` 等 HTML 标签？
- [ ] 数字列表格式是否为 `1. Text`（应改为 `1.Text` 或 `"1. Text"`）？
- [ ] 带空格的 subgraph 是否使用了 `id["Name"]` 格式？

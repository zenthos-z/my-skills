# 资源筛选提示词

日期范围限定：{{DATE}}

## JSON 输出要求（第一步）

**必须首先输出符合以下 Schema 的 JSON 数据**，然后再生成 Markdown 表格：

```json
{
  "type": "resource",
  "date_range": ["2026-04-06", "2026-04-08"],
  "resources": [
    {
      "time": "2026-04-08 06:39",
      "title": "资源标题",
      "type": "链接",
      "summary": "资源简介",
      "content": "https://...",
      "shared_by": "分享人"
    }
  ]
}
```

**字段说明**：
- `type`: 固定值 `"resource"`
- `date_range`: 日期范围数组，格式 `YYYY-MM-DD`
- `resources`: 资源数组
  - `time`: 资源发布时间，格式 `YYYY-MM-DD HH:MM`
  - `title`: 资源标题（15字以内）
  - `type`: 资源类型，枚举值：链接、paper、其他、知识文档、开源项目、网站、文章、新闻、视频、工具、论坛、技术博客、活动
  - `summary`: 资源简介（根据上下文、链接标题、讨论议题综合描述）
  - `content`: 资源链接或具体内容
  - `shared_by`: 分享人姓名

---

目标：根据要求，整理出群友讨论的所有资源，并且根据前后讨论的内容，生成以下清单

| 时间段              | 资源标题 | 资源类型 | 资源简介 | 具体内容 | 分享人 |
| ---------------- | ---- | ---- | ---- | ---- | --- |
| YYYY-MM-DD HH:MM |      |      |      |      |     |

详细说明：

时间段：资源发出时间，方便后期定位。

时间段格式：YYYY-MM-DD HH:MM

资源标题：15字以内，要求精炼的准确表达资源内容。不要用文件原名作为标题，要讲清楚核心内容。

资源类型：链接、paper、其他、知识文档、开源项目、网站、文章、新闻、新闻

资源简介：根据上下文、链接标题、讨论议题，综合描述这个资源的作用
具体内容：资源链接。如果没有则填写：手动上传

链接格式说明：表格中不要出现``​符号，用`/`隔离不同的链接

**额外说明：** 

- 大段地论文清单 表格不方便复制，在表格外单独列出来。

- 不需要列出群日报更新的消息。

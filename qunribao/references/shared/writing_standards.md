# 结论写作规范

Claude 直接读取本文件，获取完整定义。

---

## 因果链格式要求

**必须格式**：背景（为什么讨论) → 讨论(过程/测试/观点) → 结论(共识/方案)

**自检问题**：如果删掉背景/讨论，只看结论是否还能理解？
- 否 → 说明缺少因果链，需要补充

---

## 正确示例

```
| 17:28-17:38 | ⭐明确的价值议题 | 强化学习：框架能力对比 | 对比测试完成 | 背景：成员A在调研自进化论文时需要提取文献链接→讨论：用相同模型和skill分别测试OpenClaw和Claude Code提取论文信息→结论：OpenClaw更"聪明"（自主使用提取工具、提供可点击链接)，Claude Code更偏编程 | 成员A、成员B |
```

## 错误示例
```
| 17:28 | ⭐明确的价值议题 | 强化学习 | 对比测试 | OpenClaw更聪明 | 成员A |
```
↑ 缜只背景和讨论 → 结论，存的是结论缺少因果链，需要补充

---

## 结论记录的表格式说明

| 元素  | 说明           | 示例                              |
| --- | ------------ | ------------------------------- |
| 背景  | 为什么讨论这个问题    | "某成员在调研自进化论文时需要提取文献链接"            |
| 讨论  | 主要观点、争论、测试过程 | "用相同模型分别测试OpenClaw和Claude Code" |
| 结论  | 最终共识或发现      | "OpenClaw更聪明，Claude Code更偏编程"   |

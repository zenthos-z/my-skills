# 日报配图内容提炼指南

你是一位专业的内容提炼专家。请将以下日报内容精炼为适合生成信息图的简洁内容。

## 提炼原则

- **提取核心议题与观点**：保留 ⭐ 和 🔄 级别议题的标题和结论，认知洞察和行动信息同等重要

- **保留关键数据与概念**：具体数字、工具名称、技术概念、方法论框架、决策结果

- **去除冗余**：过滤讨论过程、背景描述、重复表达、因果推导细节

- **控制篇幅**：精炼内容控制在原文 15%-25%

- **结构化输出**：按议题重要程度排列

## 🚀 群聊日报提炼方法论：从信息流到认知资产

### 一、 认知差过滤机制 (The Delta Filter)
第一步是建立筛选准则。不要记录“发生了什么”，而要记录“产生了哪些独特的增量”。

* **实操深度 (Operational Truths)**：
    * **核心**：必须是“亲手做过”才能得到的结论。
    * **关注点**：具体的参数配置、工具组合的兼容性、真实测试的性能数据、未在文档中注明的“坑”或 Bug。
* **渠道不对称 (Information Asymmetry)**：
    * **核心**：AI 搜索或公开搜索引擎难以直接触达的信息。
    * **关注点**：特定的镜像站、未公开的 Skill/插件、小众但高效的工具、行业内部的资源路径。
* **群体博弈结论 (Synergistic Insights)**：
    * **核心**：多人讨论、质疑、修正后达成的共识。
    * **关注点**：针对某个痛点的最优路径选择、对某个趋势的定性判断、比单一作者文章更具辩证性的观点。
---
### 二、 价值象限重组 (Structural Reorganization)
完全打破原始记录的时间轴顺序，按**功能价值**重新编排，实现信息熵减。

| 象限模块 | 提炼逻辑 | 交付物形态 |
| :--- | :--- | :--- |
| **实战工作流 (Workflow)** | 工具与工具之间的“缝合”逻辑。 | 流程图 / 步骤列表 |
| **底层硬事实 (Hard Facts)** | 逻辑闭环后的定量描述或核心公式。 | 数学公式 / 逻辑等式 |
| **校验过的资源 (Validated Assets)** | 经过群友实测好用的工具/链接。 | 附带“实测评价”的索引 |
| **认知拐点 (Evolution)** | 讨论重心从 A 到 B 的位移，代表了学习阶段的进化。 | 趋势线 / 阶段描述 |
---
### 三、 精华提取与提纯技术 (Distillation Techniques)
通过以下手段对筛选出的内容进行“脱水”：

1.  **公式化表达**：将复杂的文字描述转化为逻辑公式。
    * *例：[目标产出] = [核心变量 A] + [核心变量 B] - [损耗因素]*
2.  **去背景化处理**：剥离具体的讨论对话背景（如“某某说”、“我想请教”），只保留结论本身。
3.  **认知补丁 (Cognitive Patch)**：不只是记录资源，要记录“为什么要用它”以及“它解决了什么特定问题”。
4.  **极简视觉化趋势**：将复杂的协作关系（如多 Agent 协作）抽象为具象的社会模型（如“虚拟公司”或“蜂群模型”）。
---
### 四、 “群体大脑”的独特溢价 (Group Brain Premium)
提炼时需刻意捕捉群聊形式独有的认知差：
* **即时反馈链**：记录从“提出问题”到“给出方案”再到“实测反馈”的完整闭环，这是静态文档不具备的动态经验。
* **集体纠偏**：记录那些被否定掉的路径（“避坑指南”），这种“失败的经验”往往比“成功的案例”更有认知价值。
* **非正式共识**：捕捉群友在讨论中流露出的“直觉”或“审美倾向”，这通常代表了该领域的最新前沿审美或技术品味。
---
## 💡 提炼效果的自检清单 (SOP)

在生成最终日报前，问自己三个问题：
1.  **脱水率是否达标？**（字数是否压缩到了原文的 10%-20%？）
2.  **认知增量是否明显？**（其中的结论是否是直接问 AI 搜不到的？）
3.  **是否具备即时调用性？**（读者是否可以不看原文，直接根据提炼内容开始行动？）
